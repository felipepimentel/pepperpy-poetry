    class SharedConfigPlugin(Plugin):
    def activate(self, poetry: Poetry, io: IO) -> None:
        """
        Activate the plugin and merge shared configurations.

        Args:
            poetry: The Poetry instance
            io: The IO instance for console output
        """
        shared_config_path = Path("shared-config.toml")

        if shared_config_path.exists():
            io.write_line("[pepperpy-poetry] Loading shared configurations...")
            try:
                shared_config = toml.load(shared_config_path)
                self._merge_configurations(poetry, shared_config)
                io.write_line("[pepperpy-poetry] Shared configurations loaded successfully")
            except Exception as e:
                io.write_error_line(f"[pepperpy-poetry] Error loading shared configurations: {str(e)}")
        else:
            io.write_line("[pepperpy-poetry] No shared-config.toml found. Skipping.")

    def _merge_configurations(self, poetry: Poetry, shared_config: Dict[str, Any]) -> None:
        """
        Merge shared configurations into the project's pyproject.toml

        Args:
            poetry: The Poetry instance
            shared_config: The shared configuration dictionary
        """
        for section, values in shared_config.items():
            if isinstance(values, dict):
                poetry.pyproject.data.setdefault(section, {}).update(values)
            else:
                poetry.pyproject.data[section] = values

    Configuration Types

    Shared Configuration

    The shared configuration is loaded from the shared-config.toml file and follows this structure:

    Dict[str, Union[Dict, List, str, int, float, bool]]

    Example: 

    [tool.ruff]
line-length = 88  # int
target-version = "py312"  # str
extend-exclude = [  # List[str]
    ".pytest_cache",
    ".ruff_cache"
]

[tool.mypy]  # Nested dict
python_version = "3.12"
strict = true  # bool

    Plugin Lifecycle

    1. Activation: The plugin is activated when Poetry starts
    2. Configuration Loading: The plugin searches for and loads the shared-config.toml file
    3. Configuration Merging: The shared configurations are merged into the project's pyproject.toml
    4. Feedback: The plugin provides console feedback about the process

    Error Handling

    The plugin handles various error scenarios:

    1. Missing Configuration File: A warning message is displayed
    2. Invalid TOML: An error message with details about the parsing error
    3. Permission Issues: An error message about file access problems
    4. Merge Conflicts: Detailed information about conflicting configurations

    Console Output

    The plugin provides feedback through Poetry's console:

    # Success case
[pepperpy-poetry] Loading shared configurations...
[pepperpy-poetry] Shared configurations loaded successfully

# Warning case
[pepperpy-poetry] No shared-config.toml found. Skipping.

# Error case
[pepperpy-poetry] Error loading shared configurations: [error details]

    Integration Points

    The plugin integrates with Poetry through:

    1. The poetry.plugins.Plugin interface
    2. Poetry's configuration management system
    3. The console I/O system for user feedback
