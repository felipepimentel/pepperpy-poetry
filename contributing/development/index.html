    Development Setup

    This guide will help you set up your development environment to contribute to the Pepperpy Poetry Plugin.

    Prerequisites

    • Python 3.12 or higher
    • Poetry installed on your system
    • Git for version control

    Setting Up the Development Environment

    1. Clone the repository: 

      git clone https://github.com/felipepimentel/pepperpy-poetry
cd pepperpy-poetry

    2. Install dependencies: 

      poetry install

    3. Install the plugin in development mode: 

      poetry self add .

    Project Structure

    pepperpy-poetry/
├── pepperpy_poetry/
│   ├── __init__.py
│   └── plugin.py
├── tests/
│   ├── __init__.py
│   └── test_plugin.py
├── docs/
│   └── ...
├── pyproject.toml
├── README.md
└── shared-config.toml

    Development Workflow

    1. Create a new branch for your feature: 

      git checkout -b feature/your-feature-name

    2. Make your changes and write tests

    3. Run the test suite: 

      poetry run pytest

    4. Format your code: 

      poetry run ruff format .

    5. Run linting: 

      poetry run ruff check .
poetry run mypy .

    6. Commit your changes: 

      git add .
git commit -m "feat: your feature description"

    Testing

    Running Tests

    # Run all tests
poetry run pytest

# Run with coverage
poetry run pytest --cov=pepperpy_poetry

# Run specific test file
poetry run pytest tests/test_plugin.py

    Writing Tests

    1. Create test files in the tests/ directory
    2. Use descriptive test names
    3. Follow the existing test patterns
    4. Include both positive and negative test cases

    Example test: 

    def test_shared_config_loading():
    # Setup
    plugin = SharedConfigPlugin()

    # Test
    result = plugin.load_config()

    # Assert
    assert result is not None

    Documentation

    Building Documentation

    # Install documentation dependencies
poetry install --with docs

# Build documentation
mkdocs build

# Serve documentation locally
mkdocs serve

    Writing Documentation

    1. Documentation is written in Markdown
    2. Place new documentation in the appropriate directory under docs/
    3. Update mkdocs.yml if adding new pages
    4. Include code examples where appropriate

    Release Process

    1. Update version in pyproject.toml
    2. Update CHANGELOG.md
    3. Create a release commit: 
      git add pyproject.toml CHANGELOG.md
git commit -m "chore: release v0.1.1"
    4. Tag the release: 
      git tag v0.1.1
    5. Push changes: 
      git push origin main --tags
